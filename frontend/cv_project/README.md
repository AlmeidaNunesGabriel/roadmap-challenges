# 📄 Currículo Online - HTML Semântico

Este projeto faz parte da minha jornada de reciclagem técnica e consolidação de fundamentos web, seguindo os desafios do [Roadmap.sh](https://roadmap.sh/frontend). 

O objetivo principal foi construir um currículo profissional utilizando as melhores práticas de **HTML5 Semântico**, foco em **Acessibilidade** e preparação para **SEO**.

## Objetivos do Projeto
- Implementar uma estrutura de dados clara e organizada.
- Utilizar tags semânticas para melhorar a indexação por motores de busca (SEO).
- Garantir que o conteúdo seja acessível para leitores de tela.
- Configurar Meta Tags de compartilhamento (Open Graph).

## Tecnologias e Conceitos Aplicados
- **HTML5 Semântico:** Uso de `<header>`, `<main>`, `<section>`, `<article>` e `<footer>`.
- **SEO & Social:** Configuração de `meta description` e `Open Graph` para visualização em redes sociais (LinkedIn/WhatsApp).
- **Acessibilidade:** Uso de atributos `alt` em imagens e hierarquia correta de títulos (`h1` a `h3`).
- **Git/GitHub:** Fluxo de trabalho via terminal (Linux/Manjaro) e GitHub CLI (`gh`).

#  Online Resume - Semantic HTML

This project is part of my technical refreshment journey and consolidation of web fundamentals, following the challenges from [Roadmap.sh](https://roadmap.sh/frontend). 

The main goal was to build a professional resume using **Semantic HTML5** best practices, focusing on **Accessibility** and **SEO** readiness.

## Project Objectives
- Implement a clear and organized data structure.
- Utilize semantic tags to improve Search Engine Optimization (SEO).
- Ensure content accessibility for screen readers.
- Configure Open Graph Meta Tags for optimized social media sharing.

## Technologies & Concepts Applied
- **Semantic HTML5:** Implementation of `<header>`, `<main>`, `<section>`, `<article>`, and `<footer>`.
- **SEO & Social:** Configuration of `meta description` and `Open Graph` for professional preview on platforms like LinkedIn and WhatsApp.
- **Accessibility:** Use of `alt` attributes for images and a correct heading hierarchy (`h1` to `h3`).
- **Git/GitHub:** Workflow managed via terminal (Linux/Manjaro) and GitHub CLI (`gh`).
